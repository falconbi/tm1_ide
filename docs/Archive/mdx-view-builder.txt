MDX View Guided Builder — Technical Summary
============================================

Overview
--------
A guided step-by-step MDX view builder within the TM1 IDE that helps users
construct SELECT/WHERE MDX queries without raw coding. It is part of a dual-mode
MDX system alongside the MDX Subset Guided Builder. Both are rendered by the
single `GuidedMDXBuilder.jsx` component with a `mode` prop derived from the
tab type.

----------------------------------------------------------------
Files & Relationships
----------------------------------------------------------------

1.  client/src/components/GuidedMDXBuilder.jsx  (main component)
    - 1000-line React component with dual modes: 'subset' and 'view'
    - Mode determined by tab.type: 'guidedmdxsubset' vs 'guidedmdxview'
    - Contains WRAPPERS (34 composable MDX pattern functions)
    - Contains PATTERNS (34 pattern definitions with categories)
    - Contains FUNCTIONS (30 individual MDX function templates with descriptions)
    - State: step, selectedCube, selectedDim, dimConfig, currentMDX,
            buildHistory, intent, timeDim/2, timeExpr1/2, etc.
    - Full help panel with search, session save/load (localStorage)

2.  client/src/components/EditorPane.jsx
    - Renders GuidedMDXBuilder for guidedmdxsubset and guidedmdxview tab types
    - Also renders SubsetEditor, ViewEditor, ChoreEditor, ProcessEditor

3.  client/src/App.jsx
    - Single toolbar icon (Braces) with dropdown: Guided Subset / Guided View
    - onClick creates tabs of type guidedmdxsubset / guidedmdxview
    - Previously had separate MDX Sandbox and Subset MDX Builder buttons (REMOVED)

4.  core/mdxBuilder.js
    - Shared buildMDX() function used by ViewEditor's MDX mode
    - Converts axis config {cube, rows, columns, pages} into SELECT statement
    - Uses TM1SubsetToSet for named subset references
    - Exported as ESM with @core alias in vite.config.js

5.  client/src/components/SubsetEditor.jsx
    - Supports tab.mdx for direct handoff from GuidedMDXBuilder
    - Visual <-> MDX toggle, Monaco editor, AI generation, validation

6.  client/src/hooks/useApi.js
    - useCubes(server)           → list of cube names
    - useCubeDimensions(server, cube) → dimensions for a cube
    - useDims(server)            → all dimension names
    - useDimAttributes(server, dim) → element attribute definitions
    - useAttributeValues(server, dim, attr) → distinct attribute values
    - useElements(server, dim)   → element members of a dimension
    - useSubsets(server, dim)    → named public subsets

7.  server.js
    - POST /api/period-builder/run   → saves TI processes
    - POST /api/process/run          → executes TI processes
    - GET  /api/dimension/attribute-values → distinct attribute values
    - POST /api/subset/preview       → runs MDX subset and returns members
    - POST /api/mdx/execute          → executes MDX query

8.  core/tm1_client.js
    - createOrReplaceProcess()  → saves TI to TM1 (delete+recreate on conflict)
    - executeProcess()          → runs TI with URL-encoded name + param array
    - previewMDX()              → creates temp MDX subset, reads elements
    - getElementAttributeValues() → reads element attribute values

----------------------------------------------------------------
Guided View Builder Flow (mode = 'view')
----------------------------------------------------------------

Step 0 — Intent + Cube Selection:
  1. User sees "What do you want to build?" with 6 intent templates:
     - Measures by Dimension
     - Cross-tab
     - Time Series (asks which dims are time, up to 2)
     - Filtered by another cube (asks for second cube)
     - Cross-cube reference
     - Freeform
  2. Each intent has detailed hover tooltip with MDX examples
  3. After intent, cube list appears; user picks a cube
  4. Intent auto-config useEffect sets up dimConfig axes automatically

Step 1 — Assign Dimensions:
  1. Shows all cube dimensions with Cols/Rows/Filt assignment buttons
  2. Per-dimension subset expression text inputs (optional)
  3. Measures dimension auto-detected (last cube dim); shows measure checkboxes
  4. Auto-config assigns axes based on intent:
     - Measures by Dimension: non-measure dims → Rows, measures → Columns
     - Time Series: selected time dim(s) + measures → Columns, rest → Rows
     - Cross-tab: first dim → Rows, second → Columns

Step 2 — Review:
  1. "Copy MDX" button
  2. "Open in Raw Editor" button

Right Panel visible on all steps:
  1. "Generated MDX" header + Run button
  2. Editable Monaco editor at 55% height for the MDX (syntax highlighted)
  3. Results grid (AG Grid) below filling remaining space

----------------------------------------------------------------
Key State Variables
----------------------------------------------------------------

step              : number   Current step (0-2)
selectedCube      : string   Cube name for view mode
selectedDim       : string   Dimension name for subset mode
dimConfig         : object   {dim: {axis, subsetExpression}} per dimension
currentMDX        : string   Current MDX in subset mode
viewMDX           : string   User-edited MDX in view mode
buildHistory      : []       Array of {id, label} for pattern layers
intent            : string   Selected intent template
secondCube        : string   Second cube for multi-cube intents
timeDim/timeDim2  : string   Selected time dimensions
timeExpr1/timeExpr2: string  Custom subset expressions for time dims
selectedMeasures  : []       Checked measure names
previewMembers    : []       Preview results in subset mode
previewResult     : object   AG Grid results in view mode

----------------------------------------------------------------
Pattern Categories (PATTERNS)
----------------------------------------------------------------

Basic, Sorting, Ranking, Filtering, Advanced, Set Ops, Hierarchy,
Time / Range, Functions (30 individual function templates)

Known Issues (as of commit 22a227b)
----------------------------------------------------------------

1. Run button disabled because viewMDX || generatedMDX condition uses
   !viewMDX for disabled check; needs !(viewMDX || generatedMDX)
2. viewMDX not synced with generatedMDX via useEffect; needs startup sync
3. Time Series intent auto-config depends on dims list (useDims) loading;
   breaks if server response is slow
4. Double curly braces in generated output: dimExpressions already
   contains braces, build() function adds a second set
   (partially fixed via * crossjoin shorthand)
5. File is 1000+ lines; dual-mode complexity makes edits fragile
